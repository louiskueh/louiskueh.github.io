C ++ => MIPS Compiler

Built entire compiler for C++ to MIPS assembly language from ground-up - Comprising of three parts :

  • Lexer - Regex
  • Parser - Bison
  • Code Generation - Mips
Lexer

The first part of any compiler is the lexer. This is used to read the input stream. For my compiler I used Flex, a lexical analyser generator. I learnt how to use regex to match certain characters, for example if the input was a number of character, as well as special characters in the C++ language 'int', 'string', 'cout'

Parser

Next I defined the grammar of our inputs, in this case C++ syntax. This was done using Bison. In order to build and parse large inputs, I used an Abstract Syntax Tree to parse the inputs in a top-down approach. One example of an expression grammer I defined in Bison is shown below :

  • Exp -> Term ExpOp Exp | Term
  • Term -> Factor TermOp Term | Factor
  • Factor -> Number FactorOp Number | Number
Code Generation

The target output was MIPS assembly language. This was, in my opinion, the hardest part of the compiler. From the parser, I would have access to the whole Abstract Syntax Tree (AST) of the inputs. The next step would be to generate assembly instructions, manually. This would be done by going through the AST and writing to the file the necessary commands, for example 'Add $reg1, $reg2, $reg3'.

This was especially hard when you consider nested expressions, such as 'int i = 1 + 2 + (3 * 4 )...etc' as well as loops and conditional statements. Lastly, I also performed some pre-processing of the C C++, defining names and recording line numbers for debugging.
